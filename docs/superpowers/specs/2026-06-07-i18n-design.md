# UI 다국어(i18n) 시스템 설계

> 상태: 승인됨 (2026-06-07). viewer.html 우선 적용, 재사용 가능한 공용 시스템.

## 목표

jarvis-web 정적 페이지의 하드코딩된 한국어 UI 문자열을 **브라우저 언어 설정**에 따라
en/ko/ja 로 표시한다. 우선 viewer.html(공개 자막 뷰어) 한정으로 적용하되, 다른 페이지
(app.html, list 등)에도 그대로 붙일 수 있는 **공용 i18n 시스템**을 만든다.

**테스트 편의:** URL 쿼리 `?lang=ja` 가 오면 브라우저 로케일보다 **우선**한다.

비목표(YAGNI): 회의 콘텐츠(자막/번역 텍스트 자체)의 번역 — 그건 jarvis 가 이미 처리한다.
i18n 은 **UI 껍데기 문자열**만 다룬다. 언어 추가는 카탈로그에 키만 더하면 되는 구조.

## 로케일 결정 (우선순위)

1. **URL 쿼리 `?lang=<code>`** — 최우선. `new URLSearchParams(location.search).get("lang")`.
   값이 지원 목록(`ko`/`en`/`ja`)에 있을 때만 채택.
2. **`navigator.language`** — 앞 2글자 prefix 로 매칭 (`ko-KR`→`ko`, `en-US`→`en`,
   `ja-JP`→`ja`).
3. **fallback `en`** — 위 둘 다 매칭 안 되면 영어.

결정된 로케일은 `document.documentElement.lang` 에 반영한다(접근성/스타일 훅).

지원 로케일: `ko`, `en`, `ja`. (자막 콘텐츠 언어에는 `zh` 도 있으나, **UI 로케일**은 en/ko/ja 만.)

## 아키텍처

### 전달 방식: 공용 `/i18n.js` 정적 자산

기존 정적 HTML 번들링 방식(`wrangler.toml` 의 Text rule 로 HTML 을 문자열 import)을
그대로 따른다. i18n 로직은 **별도 JS 파일** `src/static/i18n.js` 로 두고, 라우트
`/i18n.js` 로 서빙한다. 각 페이지는 `<script src="/i18n.js"></script>` 로 로드한다.

- **wrangler.toml**: Text rule glob 을 좁게 확장 — `globs = ["**/*.html", "**/static/i18n.js"]`.
  (`**/*.js` 로 넓히면 node_modules 의 .js 까지 텍스트 취급되므로 `**/static/i18n.js` 로 한정.)
- **src/static.d.ts**: `declare module "*/i18n.js" { const content: string; export default content; }` 추가.
- **src/index.ts**: `import I18N_JS from "./static/i18n.js"` 후
  `app.get("/i18n.js", () => new Response(I18N_JS, { headers: { "content-type": "application/javascript; charset=utf-8" } }))`.

> 왜 별도 파일인가: viewer/app/list 가 같은 카탈로그·로직을 공유해야 한다. HTML 마다
> 인라인 복붙하면 DRY 위반. 한 파일을 한 라우트로 서빙하면 페이지들이 `<script src>` 로 재사용.

### `i18n.js` 공개 API (`window.I18N`)

IIFE 로 즉시 실행, `window.I18N` 에 다음을 노출:

```js
window.I18N = {
  locale,                 // 결정된 로케일 문자열: "ko" | "en" | "ja"
  t(key, vars),           // 카탈로그 조회. 없으면 key 자체 반환. vars 로 {n} 치환.
  apply(root = document)  // root 하위의 data-i18n / data-i18n-ph 를 채움
};
```

- 카탈로그는 파일 내부 상수 `STR = { ko: {...}, en: {...}, ja: {...} }`.
- `t(key, vars)`: `STR[locale][key]` 조회 → 없으면 `STR.en[key]` → 없으면 `key` 반환.
  `vars` 객체가 오면 문자열 내 `{name}` 토큰을 `vars.name` 으로 치환(예: 재연결 카운트다운 `{n}`).
- `apply(root)`: `root.querySelectorAll("[data-i18n]")` 의 `textContent` 를 `t(key)` 로,
  `[data-i18n-ph]` 의 `placeholder` 속성을 `t(key)` 로 채운다.
- `DOMContentLoaded` 시 `apply(document)` 자동 1회 + `document.documentElement.lang = locale` 설정.

### 정적 vs 동적 문자열

- **정적**(HTML 에 고정): `data-i18n="key"` (텍스트), `data-i18n-ph="key"` (placeholder).
  `apply()` 가 DOMContentLoaded 때 채운다.
- **동적**(JS 가 런타임 생성): `I18N.t("key")` 직접 호출. 탭 라벨, 연결 상태, 카드 안내문 등.

## viewer.html 문자열 인벤토리 & 키 매핑

### 정적 (data-i18n / data-i18n-ph)

| 위치 | 현재(ko) | 키 |
|------|---------|-----|
| `<title>` (L7) | 회의 자막 | `title` |
| gate-title (L167) | 🔒 회의 자막 입장 | `gate.title` |
| gate-pw placeholder (L168) | 비번 | `gate.pwPlaceholder` |
| gate-go 버튼 (L169) | 입장 | `gate.enter` |
| `#title` 기본값 (L175) | 회의 자막 | `header.defaultTitle` |
| lockbar (L181) | ↓ 새 자막 보기 | `lockbar.newCaptions` |

> `<title>` 엘리먼트에 `data-i18n="title"` 을 부여한다. `apply()` 가 `textContent` 를
> 설정하면 `<title>` 의 경우 `document.title` 도 함께 갱신되므로 별도 처리 불필요.

### 동적 (I18N.t(...))

| 위치 | 현재(ko) | 키 | 비고 |
|------|---------|-----|------|
| 탭 "대화기록" (L232) | 💬 대화기록 | `tab.log` | 이모지는 키 값에 포함 |
| 탭 "… 요약" suffix (L232) | ` 요약` | `tab.summarySuffix` | 언어명 + suffix 조합 |
| 언어명 map (L231) | 🇰🇷 한국어 / 🇺🇸 English / 🇯🇵 日本語 / 🇨🇳 中文 | `lang.ko`/`lang.en`/`lang.ja`/`lang.zh` | **국기는 콘텐츠 언어 고정**, 이름만 UI 로케일로 |
| conn live (L305) | ● live | `conn.live` | |
| conn 종료됨 (L292) | 🛑 종료됨 | `conn.ended` | |
| conn 연결 끊김 (L294) | ⏸ 연결 끊김 | `conn.disconnected` | |
| conn 잠금 (L314) | 🔒 | `conn.locked` | 이모지뿐이라 동일하지만 키로 관리 |
| conn 재연결 (L317) | 재연결 ({n}s)… | `conn.reconnecting` | `t("conn.reconnecting",{n})` |
| 카드 회의종료 (L292) | — 회의 종료 — | `card.meetingEnded` | |
| 카드 인수 (L293) | — 새 publisher 가 채널을 인수했습니다 — | `card.taken` | |
| gate 오류 (L313) | 입장할 수 없습니다 (비번/회의 확인). | `gate.errCannotEnter` | |
| gate 빈비번 (L325) | 비번을 입력하세요 | `gate.errEmptyPw` | |

> 언어명 탭 라벨 규칙: `flag` (콘텐츠 언어별 국기, 고정) + `name` (`lang.<lg>`, UI 로케일로 번역)
> + `tab.summarySuffix` (UI 로케일로 번역). 예: UI=en, 콘텐츠=ja → `🇯🇵 Japanese summary`.
> 따라서 코드는 `names[lg]` 직접 참조 대신 `I18N.t("lang."+lg)` 로, suffix 는
> `I18N.t("tab.summarySuffix")` 로 바꾼다. `(names[lg]||lg)` 의 fallback 은 `t` 가 키 반환으로 흡수.

## 데이터 흐름

1. 브라우저가 viewer.html 로드 → `<script src="/i18n.js">` 가 동기 로드(head, 본문 스크립트보다 먼저).
2. `i18n.js` IIFE 실행: 로케일 결정 → `window.I18N` 노출.
3. `DOMContentLoaded`: `I18N.apply(document)` 가 정적 문자열 채움 + `documentElement.lang` 설정
   + `document.title` 설정.
4. viewer 본문 스크립트의 런타임 렌더(탭 빌드, 연결 상태, 카드)는 `I18N.t(...)` 호출로 문자열 획득.

> 로드 순서 주의: `/i18n.js` 는 `<head>` 에서 **일반(비 async/defer) script** 로 넣어
> `window.I18N` 이 본문 inline script 실행 전에 존재하도록 한다.

## 에러 처리

- `t(key)` 미존재 키 → en fallback → key 자체 반환(절대 throw 안 함, UI 깨짐 방지).
- `?lang=` 에 지원 안 하는 값(예 `?lang=fr`) → 무시하고 navigator/en 으로 진행.
- `navigator.language` 없음/이상 → en fallback.

## 테스트 전략

**현 상태:** jarvis-web 에는 테스트 러너가 없다(`package.json` scripts = dev/deploy/tail/typecheck 뿐).
이 작업에서 **vitest + happy-dom 를 처음 도입**한다(devDependencies 추가, `"test": "vitest run"`
스크립트, `vitest.config.ts` 의 `environment: "happy-dom"`). 향후 다른 페이지 테스트에도 재사용.

`i18n.js` 는 브라우저에 텍스트로 서빙되는 IIFE 라 ES `export` 를 쓸 수 없다. 따라서 테스트는
**파일 내용을 읽어 happy-dom 컨텍스트에서 eval** 한 뒤 노출된 `window.I18N` 을 검증하는 방식
(실제 서빙 산출물을 그대로 테스트). 각 케이스에서 `location`(search)·`navigator.language` 를
happy-dom 으로 세팅한 후 파일을 eval 한다.

핵심 케이스:
- `?lang=ja` → locale `ja` (navigator 가 ko 여도 ja 우선).
- `?lang=fr`(미지원) + navigator `ko-KR` → `ko`.
- `?lang` 없음 + navigator `en-US` → `en`.
- `?lang` 없음 + navigator `de-DE`(미지원) → `en` fallback.
- `t("tab.log")` → 로케일별 정확한 값.
- `t("conn.reconnecting", {n: 2})` → `"재접속/Reconnecting/再接続 (2s)…"` 치환.
- `t("nonexistent.key")` → `"nonexistent.key"` (throw 안 함).
- `apply()`: `data-i18n`/`data-i18n-ph` DOM 요소가 올바르게 채워짐.

## 적용 범위 (이번 작업)

- **viewer.html 만** i18n 적용 (정적 속성 부여 + 동적 `t()` 치환).
- `i18n.js` / wrangler.toml / static.d.ts / index.ts 인프라 구축(공용, 재사용 대비).
- app.html, list 등은 이번 범위 밖 — 추후 같은 `<script src="/i18n.js">` + 키 추가로 확장.
